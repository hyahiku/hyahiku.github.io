01 · Overview

What Was Investigated & Why

Business Context

As Airbnb has become a competitive platform for real estate investors, understanding what drives high ratings is a practical question for any prospective host. This analysis emphasizes the renter (host) perspective rather than the customer's — specifically, whether the flexibility options a host offers correlate with their popularity rating.

Flexibility was defined across three measurable variables: cancellation policy (flexible, moderate, strict), whether instant booking is offered, and minimum nights requirements. The hypothesis going in was that more flexible hosts would achieve higher ratings.

Data Source

The dataset was collected by insideairbnb.com, which tracks active Airbnb listings worldwide. This particular extract covers New York State with 102,599 rows and 26 columns including cancellation policy, service fees, minimum nights, review rate number, price, and instant_bookable status.

The specific scope of New York State makes this particularly relevant — one of the most competitive short-term rental markets in the world, with enormous variation in listing type, price, and host strategy.

02 · Data Preparation

Cleaning a Messy Real-World Dataset

The raw data contained several issues that needed resolving before any analysis could begin — NaN values, currency symbols embedded in numeric columns, duplicate IDs, invalid values, and outliers. Each required a different approach.

1
Fix the License Column Type Error

The license column caused a display error immediately on import. The values were not recognized as defined. Casting the column to string type at the point of import resolved the error and allowed the DataFrame to load correctly.

2
Remove Duplicates Using ID as Key

Rows were sorted by id before dropping duplicates. The id column was used rather than the host name because one host can legitimately have multiple properties — the name may validly repeat, but a property ID should be unique. Rows with id = 365 were also dropped since valid IDs are seven digits long.

3
Strip Currency Symbols from Price & Service Fee

The price and service fee columns were stored as strings with '$' and ',' symbols. Even after removing '$', casting to integer failed — the comma was still present. A two-step replace (remove '$', then remove ',') resolved the issue, and both columns were successfully cast to integer.

4
Narrow to 9 Relevant Columns

From the original 26 columns, a new DataFrame new_df was created containing only the 9 columns relevant to the analysis: id, instant_bookable, cancellation_policy, service fee, minimum nights, price, review rate number, number of reviews, and a new encoded cancellation_policy_2.

5
Identify & Remove Missing Values

A for loop iterated through all columns in new_df, appending any column with missing values to a list called problems. After dropna() was applied, the loop was run again to confirm the list was empty. The row count dropped to 99,729 — confirming removal was complete.

6
Handle Invalid & Outlier Values Per Column

Each column received targeted treatment. Negative values in minimum nights were replaced with 0 (assumed data entry errors). Review rates above 5 were capped at 5; below 1 were dropped. Outliers in number of reviews and price were identified using IQR and removed.

03 · Preliminary Analysis

Analyzing 5-Star Renters in Isolation

The first analytical phase focused on the three flexibility indicators — instant_bookable, cancellation_policy, and minimum nights — but only for renters with a review rate of 5. The goal was to identify whether highly rated renters show any pattern in the flexibility options they offer.

Initial obstacle: The original plan was to filter for listings meeting all flexibility criteria simultaneously. Out of 95,000+ entries, this combined filter returned fewer than 200 rows — too small to be representative. The approach was revised: analyze each flexibility variable individually for 5-star renters.

Instant Bookable — 5-Star Renters

The ratio of 5-star renters offering instant booking versus not was visualized as a pie chart. The result was nearly a perfect 50-50 split: 50.3% offered instant booking, 49.7% did not — indicating no preference among top-rated hosts.

Figure 1. Option to Instant Book — 5-star renters (50.3% Yes / 49.7% No)

Cancellation Policy — 5-Star Renters

The cancellation policy column was quantified by encoding the three categories as 0, 1, 2 and summing frequencies. The resulting pie chart showed an almost identical three-way split: Flexible 33.3%, Moderate 33.4%, Strict 33.3% — again, no preference among top-rated hosts.

Figure 2. Cancellation Policy Flexibility Rating — 5-star renters (33.3% / 33.4% / 33.3%)

Minimum Nights — 5-Star Renters

Unlike the previous two variables, minimum nights showed a directional signal. A boxplot was used first to evaluate the percentile distribution — revealing a range of 1 to 275, with 275 identified as an outlier exceeding the IQR boundary. A bar chart was then used to show frequency distribution, revealing a strong concentration of 5-star renters at 1–2 minimum nights.

Figure 3. Boxplot — Minimum Nights (5-star)

Figure 4. Frequency — Minimum Nights (5-star)

Preliminary finding: Of the three flexibility variables, only minimum nights showed a directional pattern — 5-star renters tend to require fewer nights. Instant bookable and cancellation policy showed no differentiation. This sets up the comparative analysis in Phase 2.

04 · Final Analysis

5-Star vs. Non 5-Star — Side by Side

To determine whether the patterns observed in 5-star renters were actually distinctive, the same visualizations were produced for all renters not rated 5 — creating a direct side-by-side comparison. A heatmap was also produced to scan for any correlations missed by the initial variable selection.

Heatmap — Broad Correlation Scan

After the preliminary flexibility analysis showed limited signal, a correlation heatmap was generated across all retained numeric columns. This revealed one strong positive correlation: price and service fee. All other pairs showed weak or no correlation — including the flexibility variables against review rate.

Figure 5. Heatmap of Columns — strong price/service fee correlation visible

Minimum Nights Comparison

The boxplots and bar charts for minimum nights were reproduced for both groups. The statistical summary confirms the slight difference: 5-star renters average 5.76 minimum nights versus 8.07 for non-5-star renters. Both groups concentrate heavily at 1–2 nights, but the non-5-star group has a longer tail toward higher minimums.

The heatmap identified a strong positive correlation between price and service fee. A linear regression was performed to visualize this relationship. The result is a near-perfect positive linear slope — confirming that service fee scales proportionally with price. Key finding: The only meaningful correlation in the dataset is between price and service fee — a structural relationship rather than an insight about host behavior. The three flexibility variables (instant booking, cancellation policy, minimum nights) show no meaningful association with review rating when comparing 5-star and non 5-star hosts side by side.

05 · Conclusion

What the Data Taught Us

Answer to the research question: Flexibility does not appear to meaningfully affect rental popularity — at least not as measured by cancellation policy or instant bookable status. Minimum nights shows a slight directional difference between 5-star and non 5-star renters, but not enough to draw a strong causal conclusion.

🔍

Start Broad, Narrow Later

The most important methodological lesson from this project: approach a new dataset with no expectations. Starting with preset assumptions about which columns matter led to a narrow initial analysis that missed the price-service fee relationship entirely. A broad heatmap scan first would have revealed that sooner.

⚠️

Null Results Are Valid Results

Finding no correlation is itself a meaningful finding. The assumption that flexible hosts get better ratings — intuitive from a business standpoint — was not supported by the data. This is a reminder that domain intuition and data evidence don't always align, and the data should win.

📐

Structure Code by Phase

Midway through the project, the code was reorganized from column-by-column grouping to phase-based grouping: cleaning → outlier removal → visualization. This improved readability and understanding of analytical progress significantly — a practice worth applying from the start of any project.

🚀

Future Directions

Adding booking date data could reveal seasonal pricing trends and yearly revenue growth — the variables most useful for predicting rental income. Exploring the neighborhood column could reveal whether location affects price or rating more meaningfully than the flexibility variables analyzed here.
